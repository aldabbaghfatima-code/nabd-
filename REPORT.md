## 1. نظرة عامة على المشروع

**اسم المشروع:** نبض (Nabd)  
**الوصف:** نظام ذكي لدعم التقييم النفسي للأطفال المتضررين في الحروب باستخدام تحليل السلوك غير اللفظي والصوت.  
**الغرض:** توفير أداة متقدمة للمعالجين النفسيين تمكّنهم من تحليل الحالة النفسية للأطفال من خلال تقنيات الذكاء الاصطناعي، مع التركيز على دمج البيانات الصوتية والبصرية واللغوية لتقديم تقييم شامل.

### الملف الشخصي للمشروع (`metadata.json`)
```json
{
  "name": "نبض - نظام التقييم النفسي للأطفال",
  "description": "نظام ذكي لدعم التقييم النفسي للأطفال المتضررين في الحروب باستخدام تحليل السلوك غير اللفظي والصوت.",
  "requestFramePermissions": ["camera", "microphone"]
}
```
- **`requestFramePermissions`**: يطلب المشروع صلاحيات الوصول إلى الكاميرا (`camera`) والميكروفون (`microphone`)، وهو أمر ضروري لميزة التحليل اللحظي (Real-time Analysis) التي تتطلب التقاط الفيديو والصوت مباشرة.

### 2.2 ملف `vite.config.ts` - إعدادات أداة البناء

```typescript
import tailwindcss from '@tailwindcss/vite';
import react from '@vitejs/plugin-react';
import path from 'path';
import {defineConfig, loadEnv} from 'vite';

export default defineConfig(({mode}) => {
  const env = loadEnv(mode, '.', '');
  return {
    plugins: [react(), tailwindcss()],
    define: {
      'process.env.GEMINI_API_KEY': JSON.stringify(env.GEMINI_API_KEY),
    },
    resolve: {
      alias: {
        '@': path.resolve(__dirname, '.'),
      },
    },
    server: {
      hmr: process.env.DISABLE_HMR !== 'true',
    },
  };
});
```

**الشرح التفصيلي:**

1. **الإضافات (Plugins):**
   - `@vitejs/plugin-react`: يُفعّل دعم React مع JSX التحويلي (Fast Refresh).
   - `@tailwindcss/vite`: يدمج Tailwind CSS v4 مباشرة في عملية البناء عبر Vite.

2. **`define`**: يعرّف `process.env.GEMINI_API_KEY` كمتغير عام متاح في الكود، ويُقرأ من ملف `.env`. يُستخدم هذا المفتاح للتواصل مع Google Gemini API.

3. **`resolve.alias`**: يعرّف الاسم المختصر `@` الذي يشير إلى جذر المشروع، مما يسمح بكتابة مسارات استيراد أقصر مثل `@/src/App` بدلاً من مسارات نسبية طويلة.

4. **`server.hmr`**: خاصية Hot Module Replacement - عند تعيين `DISABLE_HMR=true` في متغيرات البيئة، يتم تعطيل التحديث التلقائي للصفحة، وهذا مفيد في بيئة التطوير عبر الذكاء الاصطناعي لمنع الوميض أثناء التعديلات.

## 4. نظام التوجيه: `App.tsx`

```typescript
import { BrowserRouter as Router, Routes, Route, Navigate } from 'react-router-dom';

export default function App() {
  return (
    <Router>
      <Routes>
        {/* المسارات العامة */}
        <Route path="/" element={<LandingPage />} />
        <Route path="/about" element={<AboutPage />} />
        <Route path="/articles" element={<ArticlesPage />} />
        <Route path="/login" element={<LoginPage />} />

        {/* مسارات لوحة التحكم */}
        <Route path="/dashboard" element={<Dashboard />} />
        <Route path="/dashboard/real-time" element={<RealTimeAnalysis />} />
        <Route path="/dashboard/video-analysis" element={<VideoAnalysis />} />
        <Route path="/dashboard/profiles" element={<ChildProfiles />} />
        <Route path="/dashboard/settings" element={<Settings />} />

        {/* مسار احتياطي */}
        <Route path="*" element={<Navigate to="/" replace />} />
      </Routes>
    </Router>
  );
}
```

**الشرح التفصيلي:**

1. **`BrowserRouter as Router`**: يُغلف التطبيق بمكوّن التوجيه الذي يستخدم History API في المتصفح لإدارة التنقل بدون إعادة تحميل الصفحة.

2. **المسارات العامة (Public Routes):**
   - `/` → الصفحة الرئيسية (Landing Page)
   - `/about` → صفحة "عن المشروع"
   - `/articles` → صفحة المقالات والبحوث
   - `/login` → صفحة تسجيل الدخول

3. **مسارات لوحة التحكم (Dashboard Routes):**
   - `/dashboard` → لوحة التحكم الرئيسية مع الإحصائيات والرسوم البيانية
   - `/dashboard/real-time` → التحليل اللحظي بالكاميرا والصوت
   - `/dashboard/video-analysis` → تحليل مقاطع فيديو مسجلة
   - `/dashboard/profiles` → إدارة ملفات الأطفال
   - `/dashboard/settings` → إعدادات الحساب والتطبيق

4. **`<Route path="*" element={<Navigate to="/" replace />} />`**: مسار احتياطي (Fallback) - أي مسار غير معرّف يُعيد التوجيه إلى الصفحة الرئيسية مع `replace` التي تستبدل السجل في المتصفح بدلاً من إضافته.

5. **ملاحظة:** التوجيه حالياً لا يتضمن حماية (Protected Routes) - أي شخص يمكنه الوصول لصفحات لوحة التحكم مباشرة. التعليق في الكود يشير إلى أن الحماية ستُضاف في تطبيق حقيقي.

## 6. أدوات مساعدة: `utils.ts`

```typescript
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

**الشرح:**
- **`cn`**: دالة مساعدة تدمج بين مكتبتين:
  - **`clsx`**: تدمج أسماء الصفوف (class names) بشكل شرطي وذكي. مثلاً: `clsx('foo', false && 'bar', 'baz')` → `'foo baz'`.
  - **`twMerge`**: تحلّ تعارضات صناديق Tailwind. مثلاً: `twMerge('px-4 px-6')` → `'px-6'` (يأخذ الأخير).
- **الاستخدام**: تُستخدم بكثرة في المكون لدمج صنوف CSS بشكل ديناميكي حسب حالة المكون (نشط/غير نشط).

### نقاط القوة في المشروع:
1. **تصميم احترافي**: استخدام متقن لـ Tailwind CSS مع تصميم عربي RTL متكامل
2. **حركات سلسة**: تكامل مكتبة Motion لإضافة تأثيرات بصرية جذابة
3. **رسوم بيانية تفاعلية**: استخدام Recharts لعرض البيانات بشكل مرئي
4. **هيكلة نظيفة**: فصل واضح بين المكونات والصفحات والتخطيطات
5. **نظام ألوان متناسق**: ثلاث ألوان أساسية (primary, primary-dark, primary-light)

### نقاط تحتاج تطوير مستقبلي:
1. **المصادقة**: لا يوجد نظام مصادقة حقيقي (تسجيل الدخول شكلي)
2. **إدارة الحالة**: لا يوجد store مركزي (Redux, Zustand) - كل صفحة تدير حالتها فقط
3. **واجهة برمجة التطبيقات**: لا يوجد ربط مع backend حقيقي - كل البيانات وهمية
4. **الحماية**: مسارات لوحة التحكم غير محمية
5. **الاستجابة**: بعض الأقسام قد لا تكون متجاوبة بشكل مثالي على الشاشات الصغيرة
6. **إعادة الاستخدام**: بعض المكونات (مثل بطاقات الإحصائيات) تتكرر ويمكن استخراجها كمكونات مشتركة
