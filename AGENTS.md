## 1. نظرة عامة على المشروع

| الحقل | القيمة |
|-------|--------|
| **اسم المشروع** | نبض (Nabd) |
| **الوصف** | نظام ذكي لدعم التقييم النفسي للأطفال المتضررين في الحروب باستخدام تحليل السلوك غير اللفظي والصوت |
| **الإصدار** | 0.0.0 (مرحلة التطوير الأولية) |
| **نوع المشروع** | Frontend SPA (Single Page Application) |
| **المنفذ** | `localhost:3000` |
| **RTL** | نعم - عربي بالكامل (اتجاه RTL) |

| الطبقة | التقنية | الإصدار |

| بطاقات الأطفال (hover) | `whileHover` | `y: -5` رفع عند التحويم |
| النوافذ المنبثقة (خلفية) | `initial/animate/exit` | `opacity: 0→1→0` |
| النوافذ المنبثقة (محتوى) | `spring` | `scale: 0.9→1, y: 40→0` مع `damping: 25, stiffness: 300` |
| تأكيد الحذف | `AnimatePresence` | `opacity: 0→1, height: 0→auto` |
| شريط التقدم | `animate` | `width: 0→percentage` بمدة 1 ثانية |
| بطاقة "إضافة طفل جديد" | `initial/animate` | `delay: profiles.length * 0.05` |

#### التحقق
- `tsc --noEmit` نجح بدون أخطاء TypeScript
- جميع الميزات تعمل: عرض الملف، إضافة طفل، حذف طفل
- الإحصائيات تتحدث تلقائياً عند الإضافة/الحذف

## 13. ملاحظات للمطورين ووكلاء AI

### عند إضافة صفحة Dashboard جديدة:

1. أنشئ الملف في `src/pages/NewPage.tsx`
2. استخدم `DashboardLayout` كغلاف مع تمرير `title`
3. أضف الاستيراد في `src/App.tsx`
4. أضف `<Route>` في `App.tsx` قبل الـ catch-all
5. أضف عنصر في `menuItems` في `DashboardLayout.tsx`
6. استخدم نفس أنماط التصميم (حواف، ظلال، ألوان)

### عند إضافة نافذة منبثقة (Modal):

1. استخدم `AnimatePresence` من `motion/react` كغلاف
2. خلفية: `fixed inset-0 bg-black/50 backdrop-blur-sm z-50 flex items-center justify-center p-4`
3. محتوى: `bg-white rounded-[2rem] w-full max-w-3xl max-h-[90vh] overflow-y-auto shadow-2xl`
4. رأس: `bg-gradient-to-l from-primary to-teal-600 p-6/8 rounded-t-[2rem]`
5. حركة: `initial={{ opacity: 0, scale: 0.9, y: 40 }} animate={{ opacity: 1, scale: 1, y: 0 }}` مع `type: 'spring', damping: 25, stiffness: 300`
6. إغلاق عند النقر على الخلفية: `onClick={closeModal}` على الخلفية + `onClick={e => e.stopPropagation()}` على المحتوى
7. إغلاق بن нажر X: زر `absolute top-4 left-4 bg-white/20 backdrop-blur-sm rounded-xl`

### عند إضافة نموذج إدخال:

1. استخدم `useState` لكائن `formData` واحد بدل عدة `useState`
2. حقول الإدخال: `bg-slate-50 border border-slate-200 rounded-xl py-3 px-4 text-sm focus:outline-none focus:ring-2 focus:ring-primary/20 focus:border-primary`
3. عناوين الحقول: `text-xs font-bold text-slate-500 mb-2`
4. زر الإرسال: `bg-primary text-white rounded-2xl font-bold text-sm shadow-lg shadow-primary/20` مع `disabled` عند عدم اكتمال الحقول
5. تحقق من الحقول المطلوبة قبل الإرسال: `if (!field.trim()) return`

### عند التعديل على الكود:

- لا تضف تعليقات إلا إذا طُلب ذلك
- استخدم `cn()` من `src/lib/utils.ts` لدمج الصفوف
- استخدم `motion` و `AnimatePresence` للحركات
- استخدم Lucide React للأيقونات
- حافظ على تناسق RTL
- شغّل `npm run lint` قبل الاعتماد

### اصطلاحات التسمية:

- الصفحات: `PascalCase` (مثال: `ReportsPage.tsx`)
- المكونات: `PascalCase` (مثال: `DashboardLayout.tsx`)
- الدوال المساعدة: `camelCase` (مثال: `cn()`)
- الملفات: `PascalCase` للمكونات، `camelCase` للأدوات
- المسارات: `kebab-case` (مثال: `/dashboard/video-analysis`)
- الـ Interfaces: `PascalCase` (مثال: `ChildProfile`)
- دوال المُعالجة: `handle` prefix (مثال: `handleAddChild`)
- حالات النوافذ: `show` prefix (مثال: `showAddModal`)
